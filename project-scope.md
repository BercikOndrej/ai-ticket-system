## Context

Customers are regular people who purchase online courses from a website. The support team handles questions related to course access, payments, refunds, and general course-related issues.

## Problems

I get a lot of emails every day from my customers. My agents manually read, clasify, resolve the ticket and then send answer to my customers. Agents are using knowledge of repetetive questions to resolve already existing problem. This approach leads to slow, canned and no personal answers

## Solution

Build a ticket management system that uses AI - AI can read emails from customers, create tickets from emails, clasify tickets into categories and if it is possible manage them automatically based on common knowledge of repetetive questions and respond to our customers more faster. If problem cannot be manage automatically by AI agent pass it to human agent to resolve that issue.

## Features

- Receive support emails and create tickets
- Auto-generate human-friendly responses using a knowledge base
- Ticket list with filtering and sorting
- Ticket detail view
- AI-powered ticket classification
- AI summaries
- AI-suggested replies
- User management (admin only)
- Dashboard to view and manage all tickets

## Ticket statuses

- `Open`
- `Resolved`
- `Closed`

## Ticket classification

- `General question`
- `Technical question`
- `Request`
- `Refund`
